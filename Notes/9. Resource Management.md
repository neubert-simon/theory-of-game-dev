# Resource Management

##  Game Resource Types
- 3D Object Meshes and Environments (geometry)
- 3D Mesh/Object Animation Data
- Map/Level Data
    * Trigger events, object types, scripts, ...
- Sprite and Texture Data
- Sound, Music, and Recorded Dialogue
- Video and Pre-rendered Cinematics

### Important aspects
- **Storage requirements and file size**
  Different resource types require different amounts of storage. Large assets increase download size, memory usage, and loading times, so efficient storage is important.

- **File formats and scalability**
  The chosen file format affects compatibility, loading speed, quality, and whether the asset can be scaled or adapted to different platforms and hardware capabilities.

- **Compression possibilities**
  Assets can often be compressed to reduce storage space and download size. Compression may be **lossless** (no quality loss) or **lossy** (smaller files with some quality reduction), trading file size for quality and performance.


## Video and Pre-rendered Cinematics
### Pre-rendered full-motion video
> Playback of compressed video files
- **Large storage requirements**
- **Usually of higher quality**
- Can make use of real actors and scenery
- Can hide loading screens

### In-game cutscenes
> Displaying video using the game engine and renderer.
- Exported animations and camera motion
- Requires only relatively small animation files
- Audio must be treated separately
    * *can make localization easier*
- Smoother transition from/to gameplay
- **Graphics settings affect cutscene quality**
- **Can be more immersive**
    * Customizable characters work automatically
