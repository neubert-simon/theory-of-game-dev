# Hardware Components

## CPU

### Clock Rate
> The number of pulses per second generated by an oscillator that sets the tempo for the processor.

Usually measured in GHz.

### Register
> A small amount of storage available for data used for arithmetic, manipulation or testing by machine instructions.

Contains the data loaded from the Cache, on which operations are actually carried out.

### Other Units
- Control Unit: Fetches, decodes and sends instructions
- Arithmetic Logic Unit (ALU): Carries out integer math
- Floating Point Unit (FPU): Carries out floating-point math

### Cache

#### Cache lines
> Not single words, but cache lines are read if data is requested

A cache line is the **smallest unit** the machine can load from RAM into Cache.
Cache can only handle aligned memory addresses (mod cache line size).

#### Cache hit
> Data is already in the cache and can be accessed quickly
#### Cache miss
> Data is not in the cache and must be loaded from RAM

### RAM
> Data can be accessed (read or written) in almost the same amount of time irrespective of the physical location of data inside the memory.

#### Types
- **Static RAM (SRAM)**
A bit of data is stored using the state of a six transistor memory cell.
*Fast, but expensive (often used as cache)*
-  **Dynamic DRAM (DRAM)**
Stores a bit of data using a transistor and capacitor pair, which together comprise a DRAM memory cell.
*Cheaper, bur slower (often used as system memory)*

Processors have developed faster than RAM, thus memory became the bottleneck.

### CPU vs GPU
> CPUs are designed to execute a single thread as fast as possible
- Out-of-order execution
- Branch prediction
- Memory pre-fetching
- Multi-level cache hierarchy
> GPUs are designed to perform thousands of identical instructions on different data / elements in parallel
- Transforming every single vertex of a mesh to homogeneous clip space
- Calculating texture and light for millions of pixels on the screen
- Simulating thousands of independent particles
